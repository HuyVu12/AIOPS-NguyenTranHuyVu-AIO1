To run the MLOps anomaly detection pipeline from start to finish: first, spin up the Docker stack from the project root using `docker compose -f configs/docker-compose.yml up -d` (or `bash scripts/start_stack.sh`), set the tracking variable `export MLFLOW_TRACKING_URI=http://localhost:5000`, and generate the simulation datasets using `uv run python data/generate_data.py`. Next, navigate into the `huyvu/` directory, train and register the baseline production model v1 using `uv run python pipeline.py --data ../data/baseline.csv`, and launch the FastAPI server in a separate shell inside `huyvu/` using `uv run python serve.py`. Finally, run the retrain and rollback lifecycle orchestrator loop from the `huyvu/` folder using `uv run python retrain.py --reference ../data/baseline.csv --current ../data/drifted.csv --holdout ../data/holdout.csv --post-deploy-eval ../data/post_deploy_eval.csv --auto-approve --simulate-degradation` to run the stress test rollback scenario, or omit the `--simulate-degradation` flag to run correct production monitoring.
